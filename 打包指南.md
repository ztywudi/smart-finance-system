# 📦 打包EXE可执行文件指南

本指南将帮助你将"智能财务系统"打包成Windows可执行文件(.exe)。

---

## 方法一：使用PyInstaller（推荐）

### 步骤1：安装Python和依赖

1. 下载安装Python 3.10+（勾选"Add Python to PATH"）
   - 下载地址：https://www.python.org/downloads/

2. 安装项目依赖：
```bash
pip install PyQt6 openpyxl pyinstaller
```

### 步骤2：打包成EXE

进入项目目录，执行以下命令：

**简单打包（单个EXE文件）：**
```bash
pyinstaller --onefile --windowed --name "智能财务系统" main.py
```

参数说明：
- `--onefile`：打包成单个exe文件
- `--windowed`：不显示命令行窗口（GUI程序）
- `--name`：exe文件的名称

**高级打包（包含图标和额外资源）：**
```bash
pyinstaller --onefile --windowed ^
  --name "智能财务系统" ^
  --icon=resources/icons/app.ico ^
  --add-data "resources;resources" ^
  --hidden-import PyQt6.QtCore ^
  --hidden-import PyQt6.QtGui ^
  --hidden-import PyQt6.QtWidgets ^
  main.py
```

### 步骤3：获取EXE文件

打包完成后，exe文件位于：
```
dist/智能财务系统.exe
```

## 方法三：使用Nuitka（更高级的打包）

Nuitka可以将Python代码编译成真正的C代码，性能更好：

### 安装
```bash
pip install nuitka zstandard
```

### 打包
```bash
python -m nuitka --onefile --windows-disable-console --enable-plugin=pyqt6 main.py
```

## 📝 打包注意事项

### 1. 缺少模块问题

如果打包后运行提示缺少模块，使用`--hidden-import`参数：

```bash
pyinstaller --onefile --windowed ^
  --hidden-import openpyxl ^
  --hidden-import sqlite3 ^
  main.py
```

### 2. 资源文件问题

如果程序需要读取外部文件（如图标、样式表），需要打包时包含：

```bash
pyinstaller --onefile --windowed ^
  --add-data "resources;resources" ^
  --add-data "styles;styles" ^
  main.py
```

在代码中，需要动态获取资源路径：

```python
import sys
import os

def resource_path(relative_path):
    """获取资源的绝对路径"""
    if hasattr(sys, '_MEIPASS'):
        return os.path.join(sys._MEIPASS, relative_path)
    return os.path.join(os.path.abspath("."), relative_path)

# 使用示例
icon_path = resource_path("resources/icons/app.png")
```

### 3. 防病毒软件误报

打包的exe文件可能被某些杀毒软件误报为病毒。解决方法：
- 使用代码签名证书对exe进行签名
- 或者添加排除项

## 🔧 常见问题

### Q1: 打包后运行提示"Failed to execute script"

**解决方法：**
1. 使用命令行运行exe，查看具体错误信息
2. 检查是否缺少依赖模块，添加`--hidden-import`
3. 检查资源文件路径是否正确

### Q2: 打包的exe在别的电脑上无法运行

**解决方法：**
1. 确保目标电脑安装了VC++运行库
2. 使用`--onefile`模式打包
3. 或者在相同系统版本上打包

### Q3: exe文件太大

**解决方法：**
1. 使用UPX压缩：`--upx-dir=path/to/upx`
2. 使用Nuitka打包（通常更小）
3. 排除不需要的模块：`--exclude-module`
