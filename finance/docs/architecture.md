## 二、核心模块说明

### 2.1 数据库架构 (core/database.py)
- **master.db**: 主数据库，记录所有账套的元信息（账套名、会计制度、创建时间等）
- **ABxxxx_xx.db**: 每个账套独立一个数据库文件，完全隔离
- 每个账套库包含：科目表、凭证表、分录表、余额表、辅助核算表

### 2.2 会计制度管理 (core/account_system.py + accounting_systems/)
- 共注册 **11种** 会计制度
- 每种制度预置：标准科目表、报表模板、凭证分类、辅助核算类型
- 建账时选择制度 → 自动初始化对应科目体系和核算规则

### 2.3 同步引擎 (core/sync_engine.py)
- 基于时间戳的智能合并
- 支持：U盘导出 → 外出使用 → 回来源自动合并
- 冲突处理：保留最新版本 + 提示用户

### 2.4 凭证服务 (core/voucher_service.py)
- 完整凭证生命周期：草稿 → 审核 → 过账 → 结账
- 自动科目余额更新
- 借贷平衡校验

## 四、数据流

```
凭证录入 → 保存(草稿) → 审核 → 过账 → 自动更新科目余额
                                                         ↓
期初余额 → 凭证处理 → 科目余额 → 结转损益 → 生成报表 → 结账
                                           ↓
                                     Excel/PDF导出
```

## 五、U盘同步数据流

```
【办公室】                         【外出】
  主电脑                            U盘
  main.db  ←── 导出 ───→ portable.db → 插别人电脑 → 记账
              合并 ←────────           ← 回办公室插U盘
```
